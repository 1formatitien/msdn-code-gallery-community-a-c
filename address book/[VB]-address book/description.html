This is an Address Book that uses Access as a back-end.



The ADO.Net methods that i used in this example are practically
identical to their SQL counterparts, so it doesn’t take a lot of converting to
switch between OleDb + SQLClient or MySql.

 

 



I spent some time getting the names comboboxes working how i
wanted them... i.e. DISTINCT  names +
filtering so only applicable names are listed (depending on user selection in
the other combobox).



All of the database functions (connecting, disconnecting,
adding, updating, deleting, + several miscellaneous functions + methods) are
contained in the dataLayer class.



Changing any of the edit controls, triggers a check to see
if the current address has any changes, which results in the command buttons
being enabled / disabled.



If you forget to save your changes + click the new button,
you’ll be prompted to save your changes. Similarly, if you forget to save any
changes when you exit the application, you’ll be prompted to save your changes.
